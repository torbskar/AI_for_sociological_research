## Version-logging script

### Decisions
- Created `scripts/log_session_meta.sh`: captures Claude Code CLI version, configured model, git commit, and branch; appends an entry to `logs/version-log.md` only when the CLI version or model changes since the last recorded entry.
- Change-detection design chosen over append-on-every-session: produces a version audit trail without noise; each entry marks an actual version event.
- Created `logs/version-log.md` with header and first entry (Claude Code 2.1.97; default model).
- Added the script to `.claude/settings.local.json` permissions.
- Added version tracking as a new item in `outline.md` §4.6 and added `scripts/log_session_meta.sh` + `logs/version-log.md` to the supplementary materials list. Stated the known limitation: version logging captures observable identifiers but cannot detect internal model weight drift within a version string (that would require a locally deployed, pinned model).
- Updated `logs/CLAUDE.md` session-start protocol so running the script is step 1.

### Consequential reasoning
- `claude config get model` does not work as a non-interactive command on this CLI; the script falls back to the `ANTHROPIC_MODEL` env var, then to the settings files, then to `"default (see CLI version)"`. Documented in the script.
