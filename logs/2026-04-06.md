## Pipeline corrections and search term revision

### Decisions

**11 issues fixed in pipeline documentation and skill**: Three bugs that would break a future
`/lit-search` re-run (SKILL.md Steps 3 and 6 referencing Haiku sub-agents that cannot execute
Bash; Step 8 default of `--top-n 10`), five documentation errors (PIPELINE.md changelog,
boolean-searches.md header and protocol, log cross-references), one structural gap (String F
undocumented as not executed), and two cleanup items (orphaned root scripts, intermediate JSON
part files).

**Search term revision — v2 strings**: Strings A, B, C, D revised to include economics and
psychology as explicit field terms. "Academic writing" dropped from String A (student/pedagogy
noise); replaced with research-process terms. String H added (no field restriction; designed for
cross-disciplinary literature). String F remains defined but not executed.

**Venue queries added**: Five interdisciplinary journals added to search_openalex.R via ISSN
filter (Science, Nature, PNAS, Nature Human Behaviour, PLOS ONE). Rationale: papers in these
outlets do not self-identify with a discipline, so field terms in block queries structurally
exclude them.

**MAX_RESULTS raised**: 200 → 400 per query. In practice all 7 block queries still returned
exactly 200 records, suggesting the ceiling is the actual result count rather than the API
cap for these search strings.

**Download skip logic added**: download_pdfs.R now loads the existing manifest at startup and
skips previously downloaded and previously failed DOIs. Works for renamed files (check is on DOI,
not filename). Previously failed DOIs are carried forward in missing_pdfs.csv with reason noted.
