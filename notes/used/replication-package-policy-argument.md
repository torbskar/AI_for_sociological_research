# Note: Replication package logic as conservative policy argument

The replication package framing is not just a submission practicality — it is an argument for §5 (policy implications).

Journals that already require replication packages for quantitative work (Sociological Science since 2023, and many others) have already accepted the underlying principle: that the materials needed to reproduce an analysis must accompany the paper. The infrastructure exists. The norm is established.

Extending this to AI-assisted workflows requires no new principle and no new infrastructure — only an expanded understanding of what "the materials needed to reproduce the analysis" includes. For a study using regression, that means code and data. For a study using a structured AI workflow, that means skill files, prompt templates, search scripts, and screening logs. The logic is identical.

This makes the policy recommendation in §5 *conservative* rather than radical. The paper is not asking journals to adopt a new open-science norm — it is asking them to apply an existing one consistently. That is a rhetorically strong position: critics who accept replication packages for quantitative work have no principled basis for rejecting the equivalent for AI-assisted work.

## Implication for the critique of current policy

Current binary policies (permit/prohibit AI use) are not just blunt — they are inconsistent with the replication logic journals already apply elsewhere. A journal that requires statistical code but treats AI workflow documentation as optional is applying a double standard that has no methodological justification.
