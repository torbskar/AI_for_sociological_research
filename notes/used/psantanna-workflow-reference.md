**Source:** https://psantanna.com/claude-code-my-workflow/workflow-guide.html  
**Type:** Practitioner guide (not peer-reviewed). GitHub repo published with 15+ domain forks.

**What it describes:** A highly elaborate Claude Code workflow for academic research — 28 skills, 14 specialized agents, quality scoring rubrics (80/100 to commit, 95 for excellence), AEA replication standards compliance, session logging with automated stop hooks. Explicitly framed as "ceiling, not floor" with progressive entry points documented.

**Relevance to the article:**

The guide illustrates the upper end of the structured-use spectrum. It strengthens the core argument in two ways:

1. *Principle over complexity.* The epistemic move is systematic criteria and human verification at each stage — not the number of skills or agents. A minimal workflow (CLAUDE.md + a few skills + session logs) satisfies the standard just as well as an elaborate one. My own workflow being comparatively basic is a feature: it demonstrates the bar is achievable without institutional infrastructure.

2. *Progressive adoption.* PSantanna explicitly documents this as a ceiling, not a floor. Useful for preempting the objection that structured AI use is too burdensome for ordinary researchers.
